45  /**
46   * Implementation of a Product Quantiser for vectors/arrays of ints. Product
47   * Quantisers quantise data into a very large number of clusters (large enough
48   * that the centroids could not possibly fit into memory - i.e. 2^64 centroids).
49   * The Product Quantiser can be used to create compressed representations of
50   * high-dimensional vectors, and also as a means to perform efficient
51   * nearest-neighbour search over large collections of vectors (which have been
52   * effectively compressed using the product quantiser).
53   * <p>
54   * This is achieved by breaking down the input vectors into non-overlapping
55   * sub-vectors, and applying quantisation to these sub-vectors individually. The
56   * number of bins (cluster centroids) for the sub-vectors is small (up to 256 in
57   * this implementation), but when combined over all sub-vectors, the number of
58   * bins is much larger as it accounts for all combinations of bins across
59   * sub-vectors. As only a small set of centroids needs to be held for the
60   * sub-vectors, the memory requirements are quite modest. The output of the
61   * quantisation action in this implementation is an array of bytes corresponding
62   * to the index of the matching centroid for each sub-vector (index numbers are
63   * offset by -128 so that 256 centroids indexes can fit in a single byte). The
64   * bit-pattern of this byte array could be interpreted as a numeric value of
65   * global cluster index, however in practice this is not useful.
66   * <p>
67   * Typically the product quantiser is "trained" so that it adapts to the data
68   * that is is being applied too. The standard approach to this is to use
69   * K-Means, however, this is not required. Insofar as this implementation is
70   * concerned, any set of compatible {@link NearestNeighbours} implementations
71   * can be provided to the constructor. Each of the {@link NearestNeighbours}
72   * could even potentially have a different number of dimensions (corresponding
73   * to the sub-vector lengths).
74   * <p>
75   * In the standard case, where you just want to use K-Means to train the Product
76   * Quantiser, a set of utility methods can be found in the
77   * org.openimaj.knn.pq.IntProductQuantiserUtilities class which can be found in
78   * the clustering sub-project (due to the dependence on the K-Means algorithm).
79   * 
80   * @author Jonathon Hare (jsh2@ecs.soton.ac.uk)
81   * 
82   */

120 	 * Quantise the given data using this Product Quantiser. The output is an
121 	 * array of bytes corresponding to the index of the matching centroid for
122 	 * each sub-vector (index numbers are offset by -128 so that 256 centroids
123 	 * indexes can fit in a single byte).
124 	 * 
125 	 * @param data
126 	 *            the data to quantise
127 	 * @return the quantised data.
128 	 */
129 	public byte[] quantise(int[] data) {
130 		final byte[] quantised = new byte[assigners.length];
131 
132 		final int[] idx = { 0 };
133 		final float[] dst = { 0 };
134 		final int[][] qus = new int[1][0];
135 
136 		for (int i = 0, from = 0; i < assigners.length; i++) {
137 			final int to = assigners[i].numDimensions();
138 
139 			qus[0] = Arrays.copyOfRange(data, from, from + to);
140 			assigners[i].searchNN(qus, idx, dst);
141 			quantised[i] = (byte) (idx[0] - 128);
142 
143 			from += to;
144 		}
